2 Base64 is *absolutely* encryption

Base64 is an encoding scheme that represents binary data using 64 printable ASCII characters (A-Z, a-z, 0-9, +, /)

How Base64 works

1. Take 3 bytes of input ($8 \times 3 = 24$ bits)

2. Split into 4 groups of 6 bits

3. Map each 6-bit value to a Base64 character

4. Pad with = if input isn't a multiple of 3 bytes

Decode this Base64 string:

SGFybXtkM2MwZDNfdGgxc19iNHMzNjR9

Challenge 07

7 No don't worry the big ones scare me

RSA Protocol

Key generation:

1. Choose primes $p, q$. Compute $n = p \times q$

2. Compute $\phi(n) = (p-1)(q-1)$

3. Choose $e$ coprime to $\phi(n)$ (commonly $e = 65537$)

4. Compute $d = e^{-1} \bmod{\phi(n)}$ (private key)

Encryption: $c = m^e \bmod{n}$

Decryption: $m = c^d \bmod{n}$

Challenge 10

!ECDSA: Recycling, but not always

Well done on getting here! If you're playing at home, congratulations for the dedication. First, some theory:

Elliptic Curve Cryptography

An elliptic curve over a finite field $\mathbb{F}_p$ is defined by

Elliptic Curve (Short Weierstrass Form)
$$y^2 \equiv x^3 + ax + b \pmod{p}$$

The set of points $(x, y)$ satisfying this equation, together with a special "point at infinity" $\mathcal{O}$, forms an abelian group under point addition.

Point Addition

Given two points $P = (x_1, y_1)$ and $Q = (x_2, y_2)$ on the curve

Point Addition (P $\neq$ Q)
$$\lambda = \frac{y_2 - y_1}{x_2 - x_1} \bmod{p}$$ $$x_3 = \lambda^2 - x_1 - x_2 \bmod{p}$$ $$y_3 = \lambda(x_1 - x_3) - y_1 \bmod{p}$$

Point Doubling

When $P = Q$ (adding a point to itself)

Point Doubling (P = Q)
$$\lambda = \frac{3x_1^2 + a}{2y_1} \bmod{p}$$ $$x_3 = \lambda^2 - 2x_1 \bmod{p}, \quad y_3 = \lambda(x_1 - x_3) - y_1 \bmod{p}$$

Scalar Multiplication

Multiplying a point $G$ by a scalar $d$: $Q = d \cdot G = \underbrace{G + G + \cdots + G}_{d \text{ times}}$

Computing $Q = dG$ is fast, but finding $d$ given $Q$ and $G$ is the Elliptic Curve Discrete Logarithm Problem (ECDLP), which is cryptographically secure over large curves.

ECDSA Signing Algorithm

Given a private key $d$ and message $m$

1. Compute the message hash: $z = \text{SHA256}(m) \bmod{n}$

2. Choose a random nonce $k \in [1, n-1]$

3. Compute the curve point: $R = k \cdot G$

4. Extract: $r = R_x \bmod{n}$

5. Compute: $s = k^{-1}(z + r \cdot d) \bmod{n}$

6. The signature is the pair $(r, s)$

ECDSA Verification

Given public key $Q$, message $m$, and signature $(r, s)$

1. Compute $z = \text{SHA256}(m) \bmod{n}$

2. Compute $u_1 = z \cdot s^{-1} \bmod{n}$

3. Compute $u_2 = r \cdot s^{-1} \bmod{n}$

4. Compute $R' = u_1 \cdot G + u_2 \cdot Q$

5. Signature is valid iff $R'_x \equiv r \pmod{n}$

def ecdsa_sign(message: bytes):
